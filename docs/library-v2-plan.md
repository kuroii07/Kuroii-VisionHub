# VisionHub Studio 作品画廊 V2 优化计划

## 目标

作品画廊 V2 的目标是从“生成历史列表”升级为“图片管理工具”。主界面优先展示图片，搜索、过滤、排序、详情和管理操作都以低占用、按需展开的方式提供。

## 核心原则

- 图片优先：默认视图最大化展示图片，不使用常驻双列或三列详情布局。
- 详情按需：图片详情侧栏默认隐藏，只在用户点击详情时打开。
- 控制收纳：搜索、过滤、视图、排序和新增入口集中到底部悬浮搜索栏。
- 元数据安全：收藏、文件夹、收藏集、评分和显示偏好等管理信息独立保存，不破坏现有生成历史。
- 渐进迭代：先保留现有数据结构，逐步增加图片管理能力。

## V2 第一阶段范围

### 主图区域

- 使用图片优先网格作为默认视图。
- 支持视图模式：
  - 瀑布流
  - 自适应网格
  - 正方形网格
  - 完整宽高比
  - 列表视图
- 图片卡片默认只展示必要状态：
  - 缩略图
  - 收藏按钮
  - 成功 / 失败状态
  - 图生图参考标记
  - Provider / 模型信息按显示设置决定是否出现

### 详情侧栏

- 默认隐藏。
- 点击图片详情按钮后从右侧滑出。
- 关闭后主图区域恢复完整宽度。
- 详情内容：
  - 大图预览
  - Prompt
  - Provider
  - 模型
  - 尺寸、质量、数量
  - 生成耗时
  - 创建时间
  - 本地文件路径
  - 参考图来源
  - 原始 raw / 错误信息入口
  - 操作按钮：复制 Prompt、打开文件、设为参考图、删除记录

### 底部悬浮搜索栏

- 左侧是菜单按钮。
- 中间是搜索输入框。
- 右侧是加号按钮。
- 页面底部预留安全空间，避免搜索栏遮挡最后一排图片。

搜索范围：

- Prompt
- Provider
- 模型
- 收藏状态
- 文件夹 / 收藏集
- 文生图 / 图生图
- 成功 / 失败
- 日期范围
- 本地文件名 / 路径
- 错误信息

菜单项：

- 隐藏搜索栏
- 显示过滤器
- 网格样式
- 显示设置
- 排序方式

网格样式：

第一阶段可以先做菜单入口和占位提示，后续逐项接真实功能。

## 收藏与元数据

收藏功能不直接写入生成历史记录，建议新增独立元数据文件。

建议文件：

```text
library-meta.json
```

建议按 `recordId` 或本地图片路径保存：

- favorite: 是否收藏
- folderId: 所属文件夹
- collectionIds: 收藏集
- rating: 手动评分
- lastViewedAt: 最近查看时间
- lastUsedAsReferenceAt: 最近作为参考图使用时间

这样可以避免生成历史结构变化时丢失人工整理信息。

## 路线取舍原则

后续路线只保留高频、可感知、能明显提升真实工作流的功能。视觉收益低、解释成本高、占用主界面或详情空间的能力，不进入默认路线；确实需要时再作为隐藏式工具、批处理工具或开发者诊断入口单独评估。

## 分阶段实施

### 2026-06-09 第一批：桌面端元数据文件

目标：

- 先把收藏、评分、文件夹、收藏集、显示设置、最近查看和最近作为参考图使用时间从纯 renderer 缓存升级为桌面端独立数据文件。
- 不改 `generation-history.json` 结构，不删除磁盘图片，不迁移 Provider 凭据。

已落地：

- 新增桌面端 `library-meta.json` 数据层，位置在应用数据目录下，与 `generation-history.json` 分离。
- Tauri 后端新增 `load_library_data` / `save_library_data`，用临时文件替换方式写入，降低写坏元数据文件的风险。
- 前端启动作品画廊时优先读取 `library-meta.json`；如果文件不存在，会把现有 localStorage 里的画廊整理信息安全写入该文件。
- 前端后续变更继续保留 localStorage 作为本地兜底，同时 debounce 同步到 `library-meta.json`。

后续继续：

- 增加更完整的导入去重、图片尺寸读取和批量管理。
- 优先补齐用户高频使用的整理动作，例如批量移动、批量收藏、复制 Prompt / 路径、导出记录清单。

### 2026-06-09 `0.2.6-b` 批量整理 V1

已落地：

- 多选操作条从“多选后才出现”调整为选中 1 项即可出现，便于用户确认当前选择和继续批量整理。
- 批量操作补齐高频动作：批量收藏、取消收藏、复制 Prompt、复制路径、导出 Markdown 记录清单。
- 单张图片的更多菜单和右键菜单同步增加复制路径、导出清单，避免只能进入详情页操作。
- 导出清单弹出系统保存位置选择窗口；只导出记录文本信息，不移动、不删除、不打包用户图片。

### 2026-06-09 视觉复查调整：移除低价值详情块

已调整：

- 视觉复查发现“记录健康”和“标签与备注”会挤压详情抽屉，顶部健康筛选也让筛选栏变得拥挤。
- 已移除可见的记录健康、健康筛选、卡片健康标记、标签与备注编辑区。
- 保留第一批 `library-meta.json` 数据层基础；“记录健康”“标签与备注”不再作为后续路线项。

### 阶段 1：画廊 V2 基础体验

- 加入底部悬浮搜索栏。

- 多选和批量操作。
- 右键菜单。

- 批量复制 Prompt / 路径。

## 暂不做

- 不在第一阶段重做整个历史数据结构。
- 不默认删除磁盘图片。
- 不默认引入复杂数据库。
- 不把详情侧栏做成常驻布局。
- 不让底部搜索栏承载过多筛选项，复杂筛选放到菜单弹层。

## 验收标准

- 默认进入画廊时，主视觉是图片网格。
- 不打开详情时，页面没有常驻右侧详情栏。
- 搜索栏位于底部居中，不遮挡最后一排图片。
- 菜单按钮能收纳过滤、视图、显示设置、排序和缩放入口。
- 收藏状态可见、可切换、可搜索或过滤。
- 详情抽屉打开和关闭不破坏主网格的浏览体验。


### 2026-06-22 `0.4.0` 画廊整理收口方向

- 独立“项目资产库”页面不再作为主路线；作品归类继续由画廊的文件夹、收藏集、收藏、评分和筛选承担。
- 状态筛选改为明确的成功记录 / 失败记录 / 全部记录，避免“默认状态”的语义含糊。
- 打开详情或预览图片时写入 `lastViewedAt`；设为参考图时继续写入 `lastUsedAsReferenceAt`，并提供对应分类 / 排序入口。
- 本轮只做语义、元数据和路线清理，不重做已确认符合要求的按钮大小、位置、UI 和排版。
