# VisionHub Studio 0.2.5 AI 创作功能区 V3 详细计划

## 目标

`0.2.5 AI 创作功能区 V3` 聚焦 AI 创作页底部三个高频弹窗：提示词润色、复用记录、模板灵感。目标是把它们从“查看 / 回填”升级为“可编辑、可复用、可管理”的创作辅助区。

本阶段不改 Provider 凭据、不改生图协议主链路、不迁移 Provider profile id、不重置用户本地数据。

## 当前代码入口

- AI 创作页：`src/ui/GeneratePage.tsx`
- 三类弹窗：`src/ui/PromptAssistModal.tsx`
- 提示词规则与模板：`src/services/promptAssist.ts`
- 旧模板存储服务：`src/services/promptTemplates.ts`
- 全局样式：`src/ui/styles.css`
- 验证脚本：`scripts/smoke_check.py`

## 现状判断

### 已具备

- `GeneratePage.tsx` 已统一通过 `PromptAssistModal` 打开三类弹窗。
- `onApplyPrompt(prompt, placement)` 已支持替换与追加。
- `PromptAssistModal.tsx` 已包含：
  - `InspirationPanel`
  - `PolishPanel`
  - `ReusePanel`
  - `PromptPreview`
  - `AssistActions`
- 复用记录已能按关键词搜索，并支持复用、追加、复制。
- 模板灵感已支持字段填写、Prompt 预览、回填和复制。
- 润色窗口已支持本地规则和模型润色配置切换。

### 主要缺口

- 润色窗口的“原提示词”和“润色结果”仍是只读预览，不能直接编辑。
- 润色窗口缺少“恢复原文 / 交换左右 / 编辑后回填”的多轮编辑能力。
- 复用记录缺少平台、模型、文生图 / 图生图、有参考图、状态和时间排序筛选。
- 复用记录缺少删除记录入口；删除必须只删软件记录，不默认删除磁盘图片。
- 模板灵感数量仍偏少，缺少用途筛选、收藏常用模板、最近使用模板。
- 三类弹窗样式需要统一按钮层级、字符统计、空状态、长文本防溢出、浅色 / 暗色模式。

## 实施拆分

### 0.2.5-a 润色窗口 V2

优先级：最高。

任务：

- 将原提示词预览改为可编辑文本区。
- 将润色结果预览改为可编辑文本区。
- 模型润色时使用当前弹窗内编辑后的原提示词，而不是只读的主输入框文本。
- 本地润色预览随“原提示词 / 润色模式 / 风格预设”变化。
- 保留用户手动编辑过的润色结果，避免切换小状态时意外覆盖。
- 增加操作：
  - 替换当前 Prompt
  - 追加到当前 Prompt
  - 复制结果
  - 恢复原文
  - 交换原文与结果

验收：

- 用户可在弹窗内完成“输入 → 润色 → 手动改 → 回填”。
- 字符数、标题和按钮在长 Prompt 下不跳位。
- 模型润色失败时不会清空用户已编辑内容。

### 0.2.5-b 复用记录 V2

优先级：高。

任务：

- 增加筛选条：
  - 关键词
  - 平台 / Provider
  - 模型
  - 文生图 / 图生图
  - 有参考图
  - 只看成功 / 包含失败
- 增加排序：
  - 最近生成
  - 最早生成
  - Prompt 长度
- 增加记录操作：
  - 复用到当前提示词
  - 追加到当前提示词
  - 复制 Prompt
  - 删除软件记录
- 删除记录必须走现有数据层或新增最小必要接口，只删除记录，不删除磁盘图片。

验收：

- 用户可以快速定位历史 Prompt。
- 删除记录前有确认提示。
- 删除记录不影响磁盘图片文件。

### 0.2.5-c 模板灵感 V2

优先级：中高。

任务：

- 扩充模板库，至少覆盖：
  - 商业海报
  - 角色设定
  - 电商主图
  - 产品摄影
  - 游戏资产
  - 社媒封面
  - 图生图改写
  - 中文平台常用描述
- 增加用途筛选。
- 增加“替换 / 追加 / 复制”的一致操作区。
- 增加模板适合场景说明。
- 收藏常用模板和最近使用模板可作为后续增强；若实现复杂，可先做本地状态级 MVP，不引入后端迁移。

验收：

- 模板数量明显多于旧版。
- 用户能按用途筛选并快速回填。
- 文生图 / 图生图场景都有可用模板。

### 0.2.5-d 统一弹窗体验与视觉验收

优先级：贯穿全阶段。

任务：

- 统一三类弹窗的信息结构：
  - 顶部标题
  - 说明文字
  - 筛选 / 控制区
  - 主内容区
  - 底部操作区
- 统一组件状态：
  - 空状态
  - 加载状态
  - 错误提示
  - 禁用态
  - 复制成功反馈
- 统一浅色 / 暗色模式样式。
- 避免继续在 `styles.css` 到处追加零散规则；优先整理 `promptAssist*` 相关样式块。

验收：

- 三类弹窗在浅色 / 暗色模式下按钮可达、层级清晰。
- 长 Prompt 不撑破弹窗，不遮挡底部按钮。
- 图标按钮具备 `title`、`aria-label` 或清晰文本。

## 建议落地顺序

1. 先做 `0.2.5-a 润色窗口 V2`，因为它直接影响当前创作效率，且改动范围集中在 `PromptAssistModal.tsx` 和 `styles.css`。
2. 再做 `0.2.5-c 模板灵感 V2`，主要扩充 `promptAssist.ts` 和筛选 UI。
3. 最后做 `0.2.5-b 复用记录 V2`，因为删除记录可能需要数据层接口，风险略高。
4. 每完成一个小阶段运行：

```powershell
npm.cmd run build
python .\scripts\smoke_check.py
```

如改到 `src-tauri`，补跑：

```powershell
cargo check
```

## 第一批建议实现范围

第一批只做润色窗口 V2 的可编辑闭环：

- 可编辑原提示词。
- 可编辑润色结果。
- 本地 / 模型润色使用弹窗内原提示词。
- 替换、追加、复制、恢复原文、交换左右。
- 调整润色弹窗布局和长文本样式。
- 跑 `npm.cmd run build` 和 `python .\scripts\smoke_check.py`。

这批不碰 Provider 凭据、不碰生成协议、不碰历史记录删除，风险最低，也最符合 `0.2.5` 的起步目标。
