# Kuroii VisionHub v0.5.4 Release Notes Draft

Date: 2026-07-08
Status: validated local draft

## Summary

`0.5.4` is a startup performance hotfix. The observed slow release launch was traced to a 202MB `generation-history.json` containing reference image base64 payloads, plus startup-time base64 hydration of local generated images.

## Changes

- Compact generation history records before persistence.
- Remove large `data:image` payloads from persisted reference images when `local_path` is available.
- Cache embedded reference images to AppData `reference-images/` only when no local path exists.
- Stop `load_generation_history` from converting every local generated image into base64 at startup.
- Use Tauri `convertFileSrc` in the frontend to display local generated images and reference thumbnails.
- Backend generation requests now resolve references from `local_path` when no data URL is present.

## Validation

- [x] `npm.cmd run verify`
- [x] `npm.cmd run tauri:build`
- [x] Release launch smoke
- [x] History JSON compaction evidence: `generation-history.json` shrank from 211,896,487 bytes / 202.08MB to 293,235 bytes / 0.28MB; records stayed at 94; `data:image` count became 0.
