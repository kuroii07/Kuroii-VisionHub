# Kuroii VisionHub v0.5.3 Release Notes Draft

Date: 2026-07-08
Status: validated local draft

## Summary

`0.5.3` is a small naming polish hotfix after the Kuroii VisionHub branding and WebView2 stability work. It restores the Chinese product subtitle for zh-CN users and makes Tauri release builds also emit a direct user-facing portable executable copy named `Kuroii VisionHub.exe`.

## Changes

- Restored zh-CN sidebar subtitle to `AI 图片工作流工作台`.
- Kept en-US sidebar subtitle as `AI Image Workflow Studio`.
- Kept internal Cargo package name / canonical binary `visionhub-studio.exe` for build compatibility.
- Added `scripts/copy_release_exe.ps1`, called by `npm.cmd run tauri:build`, to copy the canonical release exe to `src-tauri/target/release/Kuroii VisionHub.exe`.
- Preserved Tauri identifier `studio.visionhub.app` and keyring service `visionhub-studio`, so existing AppData, gallery/history, Provider profiles, and credentials remain available.

## Compatibility notes

- Seeing `visionhub-studio.exe` under `src-tauri/target/release` is expected: it is the internal Cargo binary name.
- Use `Kuroii VisionHub.exe` or the MSI / NSIS installer for user-facing manual tests.
- No AppData, gallery, history, profile id, or credential storage migration is performed in this hotfix.
