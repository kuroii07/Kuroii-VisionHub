# VisionHub Studio v0.5.0 Release Notes Draft

Status: draft / release-candidate preparation  
Date: 2026-07-07

## Release goal

`0.5.0` is the release-candidate preparation checkpoint. The goal is not to add a new large feature, but to make the current local-first AI image workspace easier to verify, migrate, rebuild, and package.

## Included so far

- Version consistency moved to `0.5.0` across package metadata, Tauri metadata, Cargo metadata, Cargo lock, app display, README, and roadmap docs.
- Added `scripts/release_candidate_check.py` for read-only release-candidate checks:
  - version consistency across metadata and docs;
  - tracked repository hygiene for build outputs, private files, installer artifacts, archive artifacts, and high-risk secret-like literals.
- `scripts/run_checks.ps1` is now the single local verification entry point after loading the portable toolchain:
  - release candidate consistency check;
  - smoke check / UI QA baseline;
  - TypeScript + Vite production build;
  - Cargo check;
  - `git diff --check`.
- Green release-candidate build has been produced with `npm.cmd run tauri:build`.
- The release exe was launched locally from `src-tauri/target/release/visionhub-studio.exe` and the running process path was confirmed.
- Local RC artifact sizes and SHA256 values are recorded below. Build outputs and installer artifacts remain outside source control.

## Main product baseline inherited from 0.4.x

- Chinese / English i18n V1 is closed for the main UI shell and core workflows.
- Provider settings, AI Create Desk, gallery workflows, inspiration workflows, prompt tools, local model first slices, migration guide export, and global QA baseline are available as the current development baseline.
- User prompts, model names, provider names, raw API errors, file paths, and user-saved templates remain untranslated by design.

## Not included yet

- Final GitHub Release notes and Release Asset upload.
- Code signing; any local Windows build remains unsigned unless a signing certificate is added later.
- Full installer install / uninstall QA and screenshot-based visual QA.
