# Kuroii VisionHub v0.5.5 Release Notes Draft

Date: 2026-07-08

## Summary

`0.5.5` is a follow-up gallery performance hotfix after `0.5.4`.

`0.5.4` fixed the oversized startup history payload. Local follow-up diagnosis found the remaining lag was caused by the gallery still rendering and decoding too many full-size local PNG files as thumbnails.

Observed local data:

- `generation-history.json`: 0.28MB after 0.5.4 compaction.
- Records preserved: 94.
- Local generated image paths: 78 existing image files.
- Local generated image total size: about 243.48MB.
- Largest observed local PNG: 11.64MB.

## Changes

- Reduced gallery initial thumbnail render count from 48 to 18.
- Replaced next-frame automatic full-list expansion with incremental scroll/manual loading.
- Added a load-more control so users can explicitly continue thumbnail loading when needed.
- Moved gallery color analysis from the image load hot path to browser idle time.
- Kept existing AppData, gallery images, history records, Provider profile ids, and credential channels unchanged.

## Compatibility

- Tauri identifier remains `studio.visionhub.app`.
- Cargo package / internal binary name remains `visionhub-studio`.
- User-facing product name remains `Kuroii VisionHub`.
- Keyring service and secret ids remain unchanged.

## Validation

- `npm.cmd run verify` - passed
- `npm.cmd run tauri:build` - passed
- Release exe launch smoke with the existing local 94-record / 243MB generated-image library - passed: process `Kuroii VisionHub`, 14-second sample, CPU 1.19, working set 71.39MB; history remained 0.28MB with 0 `data:image` / 0 `;base64,` markers.
