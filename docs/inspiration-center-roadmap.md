# VisionHub Studio 灵感中心规划

> 本文档用于约束后续「灵感中心」的产品设计、数据结构与迭代路线。后续开发灵感相关功能时，默认以本文档为方向基准。

## 1. 功能定位

「灵感中心」是 VisionHub Studio 的创作前参考资料库，解决两个问题：

1. 管理优秀的国内外提示词模板、AI 图片案例、模型社区、风格参考网站。
2. 管理用户自己收藏的优秀 AI 图片、参考图、来源链接、Prompt、标签和备注。

它不是作品画廊，也不是单纯提示词库：

- **作品画廊**：保存自己在 VisionHub 里生成过的历史结果。
- **提示词库**：保存可以直接复用的 Prompt 模板。
- **灵感中心**：保存外部灵感来源、参考图、案例图、收藏素材与创作线索。

## 2. 导航位置

左侧导航建议顺序：

1. AI 创作
2. 免费平台
3. 作品画廊
4. **灵感中心**
5. 提示词库
6. 平台接入
7. 偏好设置

原因：灵感中心与作品画廊关系最接近，都是图片与创作资料管理；但它面向「生成前参考」，作品画廊面向「生成后结果」。

## 3. 页面信息架构

灵感中心内部使用两个 Tab，而不是拆成两个左侧菜单项：

### 3.1 灵感源

用于管理网站、社区、提示词模板站、AI 图片案例站。

核心字段：

- 名称
- URL
- 类型
  - 提示词模板
  - AI 图片案例
  - 模型社区
  - 风格参考
  - 商业设计参考
  - 其它
- 地区
  - 国内
  - 国外
  - 不限
- 标签
- 备注
- 是否需要登录
- 是否适合商用参考
- 创建时间
- 最近打开时间

核心操作：

- 新增灵感源
- 编辑灵感源
- 删除灵感源
- 一键打开网站
- 复制链接
- 搜索 / 标签筛选 / 类型筛选 / 地区筛选

第一版不做自动爬取，不模拟网页操作，只做可配置的灵感网站收藏夹。

### 3.2 图片收藏

用于管理用户收集的优秀 AI 图片、参考图、案例图。

核心字段：

- 标题
- 本地图片路径
- 缩略图路径
- 来源 URL
- 来源平台
- 作者 / 账号，可选
- 原始 Prompt，可选
- 反推 Prompt，可选，后续能力
- 标签
- 备注
- 版权 / 使用备注
  - 仅参考
  - 可商用，用户确认
  - 授权未确认
- 创建时间
- 更新时间

核心操作：

- 上传本地图片
- 拖拽导入图片
- 从剪贴板粘贴图片
- 搜索 / 标签筛选 / 平台筛选 / 授权状态筛选
- 大图预览
- 编辑标题、标签、来源、Prompt、备注
- 删除收藏记录
- 一键「作为参考」跳转 AI 创作页并进入图生图
- 一键复制 Prompt
- 一键转为提示词模板

## 4. MVP 第一版范围

第一版目标是「轻量但真实可用」，不要一次塞入复杂自动化。

### 4.1 灵感源 MVP

必须支持：

- 新增 / 编辑 / 删除灵感源
- 打开外部链接
- 复制链接
- 类型、地区、标签、关键词筛选
- 本地持久化保存

可以预置少量默认分类，但默认网站列表可以先为空，避免引入授权和推荐维护问题。

### 4.2 图片收藏 MVP

必须支持：

- 选择本地图片导入
- 拖拽导入
- 剪贴板图片导入
- 缩略图网格展示
- 大图预览，复用现有预览组件
- 标题、标签、来源 URL、Prompt、备注编辑
- 删除收藏记录
- 一键作为图生图参考
- 本地持久化保存

第一版暂不做：

- AI 反推提示词
- 联网抓取网页图片
- 自动识别作者和来源
- 图片相似度检索
- 多设备同步
- 团队共享

## 5. 与现有功能联动

### 5.1 与 AI 创作联动

图片收藏卡片提供「作为参考」按钮：

1. 点击后把收藏图加入 AI 创作页参考图列表。
2. 自动跳转 AI 创作页。
3. 自动切换到图生图模式。
4. 遵守现有最多 4 张参考图规则。

### 5.2 与提示词库联动

图片收藏卡片提供「转为模板」按钮：

1. 如果收藏图有原始 Prompt 或反推 Prompt，可以保存到提示词库。
2. 默认模板标题使用收藏图标题。
3. 标签可以继承收藏图标签。
4. 用户确认后再写入提示词库。

### 5.3 与作品画廊边界

不要把灵感图片混入作品画廊历史记录。

- 作品画廊记录 VisionHub 生成结果。
- 灵感中心记录用户主动收藏/导入的外部参考。
- 两者可以互相引用，但数据表和文件夹分开。

## 6. 本地数据与目录建议

建议独立于生成历史保存：

```text
VisionHub AppData/
  inspirations/
    inspiration-sources.json
    inspiration-assets.json
    images/
    thumbnails/
```

### 6.1 inspiration-sources.json

建议结构：

```ts
interface InspirationSource {
  id: string;
  name: string;
  url: string;
  category: 'prompt-template' | 'image-gallery' | 'model-community' | 'style-reference' | 'commercial-design' | 'other';
  region: 'china' | 'global' | 'mixed';
  tags: string[];
  note?: string;
  requiresLogin?: boolean;
  commercialReference?: 'unknown' | 'reference-only' | 'user-confirmed';
  createdAt: string;
  updatedAt: string;
  lastOpenedAt?: string;
}
```

### 6.2 inspiration-assets.json

建议结构：

```ts
interface InspirationAsset {
  id: string;
  title: string;
  imagePath: string;
  thumbnailPath?: string;
  sourceUrl?: string;
  sourcePlatform?: string;
  author?: string;
  originalPrompt?: string;
  inferredPrompt?: string;
  tags: string[];
  note?: string;
  licenseStatus: 'unknown' | 'reference-only' | 'commercial-confirmed';
  createdAt: string;
  updatedAt: string;
}
```

### 6.3 文件策略

- 导入图片时复制一份到 `inspirations/images/`，避免原始文件移动后失效。
- 缩略图后续可生成到 `inspirations/thumbnails/`。
- 删除收藏记录时，默认只删除记录；是否删除本地复制图片可以后续做成选项。
- 不保存 API Key 或用户账号信息。

## 7. UI/UX 方向

整体延续当前 VisionHub Studio 风格：

- 干净、克制、少渐变
- 纯色卡片 + 描边
- 信息密度适中
- 卡片尺寸统一
- 不用大段说明污染主界面
- 空状态简洁明确
- 浅色 / 深色模式同时考虑

### 7.1 页面布局建议

顶部：

- 标题：灵感中心
- 统计胶囊：灵感源数量、收藏图片数量、最近新增
- 主要按钮：新增灵感源、导入图片

中部：

- Tab：灵感源 / 图片收藏
- 搜索框
- 筛选器：类型、地区、标签、授权状态

内容区：

- 灵感源使用列表卡片或紧凑网格。
- 图片收藏使用统一缩略图卡片。
- 图片卡片操作按钮悬停出现，避免界面拥挤。

### 7.2 空状态

灵感源空状态：

- 主文案：还没有灵感源
- 操作：新增一个网站

图片收藏空状态：

- 主文案：还没有收藏图片
- 操作：导入图片 / 拖拽图片到这里

## 9. 风险与约束

### 9.1 版权与合规

灵感中心会保存用户从外部收集的图片，所以必须明确：

- 默认仅作为参考资料管理。
- 不默认声明可商用。
- 提供授权状态字段，让用户自己标注。
- 后续如果做推荐网站预置，需要避免误导用户以为素材可直接商用。

### 9.2 外部网站维护

不要依赖外部网站结构，不做爬虫 MVP。

第一版只保存链接和备注，这样稳定、低风险、维护成本低。

### 9.3 数据膨胀

图片收藏会占用本地磁盘，需要后续支持：

- 查看占用空间
- 打开灵感图片目录
- 清理无效记录
- 导出备份

## 10. 第一版验收标准

灵感中心第一版完成时，应满足：

- 左侧导航能进入灵感中心。
- 灵感源可以新增、编辑、删除、打开、复制链接。
- 图片收藏可以导入、预览、编辑基础信息、删除记录。
- 收藏图片可以一键作为 AI 创作参考图。
- 数据重启应用后仍然存在。
- 浅色 / 深色模式可读。
- 空状态、失败状态、长标题、长 Prompt、长 URL 不破版。
- 不把灵感图片混入作品画廊历史记录。
- 不提交用户收藏图片、AppData、密钥或本地私有路径到 Git。

## 11. 开发优先级建议

下一次正式开发建议按这个顺序：

1. 定义 `InspirationSource` / `InspirationAsset` 类型。
2. 新增 Tauri 后端读写 inspiration JSON 的命令。
3. 新增前端 service：`inspirationApi.ts`。
4. 新增 store 状态：读取、添加、更新、删除灵感源和收藏图。
5. 新增 `InspirationPage.tsx`。
6. 打通图片导入和本地复制。
7. 打通「作为参考」到 AI 创作页。
8. 补齐样式、空状态和筛选。
9. 跑检查、构建、重启验证。
10. 提交并推送 GitHub。
